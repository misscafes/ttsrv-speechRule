============================================================
猫剪豆问（优化版）vs 猫箱-VV（完全版+1）详细对比报告
============================================================

【一、文件结构变化】
  参考版(猫箱-VV): 2个对象（朗读脚本 + 音效替换规则）
  优化版(猫剪豆问): 3个对象（朗读脚本 + 音效替换规则 + 情绪桥接模块）
  => 新增第3个对象：情绪桥接(本地关键词+VV情绪引擎) v2

【二、主脚本新增全局变量】
  + var LQ
  + var PENDING_QUOTE_FILE
  + var PENDING_QUOTE_MAX_AGE
  + var RQ
  + var SIMILARITY_THRESHOLD

【三、主脚本新增函数】
  + function analyzeOrphanQuotes()
  + function clearPendingQuote()
  + function getVoiceMarkFromMatch()
  + function lcsLength()
  + function lcsSimilarity()
  + function readPendingQuote()
  + function tryMatchExact()
  + function tryMatchSimilar()
  + function writePendingQuote()

【四、主脚本修改/重命名函数】
  ~ tryMatch() -> 拆分为 tryMatchExact() + tryMatchSimilar()

【五、核心功能模块分析】

  5.1 引号悬置处理机制（新增）
    - PENDING_QUOTE_FILE: 悬置引号持久化文件
    - LQ/RQ: 左右弯引号 Unicode 字符
    - SIMILARITY_THRESHOLD: 相似度阈值 0.85
    - PENDING_QUOTE_MAX_AGE: 最大存活时间 300000ms (5分钟)
    - analyzeOrphanQuotes(): 分析孤立引号并跨段落拼接对话
    - readPendingQuote()/writePendingQuote()/clearPendingQuote(): 读写清理

  5.2 缓存匹配机制升级
    - tryMatch() -> tryMatchExact() + tryMatchSimilar()
    - 新增 lcsLength() / lcsSimilarity(): 最长公共子序列相似度计算
    - 支持精确匹配 + 模糊匹配双重策略

  5.3 发音人标记提取优化
    - 新增 getVoiceMarkFromMatch(): 从匹配结果中提取发音人标记

  5.4 情绪桥接模块（独立对象，约256行代码）
    - ENABLE_EMOTION_BRIDGE: 总开关
    - ENABLE_LOCAL_EMOTION_CORRECTION: 本地关键词校正开关
    - ENABLE_CROSS_DIALOGUE_EMOTION_INHERIT: 跨对话情绪继承开关
    - EMOTION_INHERIT_MAX_AGE_MS: 情绪继承最大时间 5分钟
    - EMOTION_MAP: 中文情绪->VV引擎情绪的映射表
    - normalizeEmotionDebugValue(): 情绪值标准化
    - extractEmotionFromTag(): 从标签提取情绪
    - applyEmotionToSettings(): 应用情绪到发音设置
