### 猫剪豆问 v1.10 修复跳段/半句话问题（2026-06-24）
- **版本升级**: v1.0.9 → v1.10（脚本 + 引擎）
- **问题背景**: 用户反馈长文本/整段旁白朗读时容易出现跳段、只读半句话、音频截断等问题。排查发现：
  1. `splitLongText` 在子句仍超过 `maxChars` 时直接按字数硬切，可能把词组或语义单元切到中间，导致半句话或语义断裂
  2. `wsUrl` / `aid` / `device_id` 在每段外层只生成一次，重试循环里一直复用同一个连接标识，容易因单标识被限流/连接复用冲突而连续失败
  3. `minAudioBytes` 固定为 200，对较长文本过于宽松，短音频/截断音频被误判为成功
  4. `MAX_CHARS_PER_SYNTHESIS = 500` 偏保守，单次合成容量不足；`TIMEOUT_MS = 30000` 在网络波动时容易超时
  5. 重试间隔线性增长，退避不够柔和
- **改动内容**:
  1. 引擎 `splitLongText` 重写为四层语义切分：
     - 第一层按句子结束符 `。！？…` 切分
     - 第二层按子句标点 `，、；：` 切分
     - 第三层按空格/连字符/中英数边界切分
     - 第四层仅在单片段无合适边界时才按字数硬切
  2. 将 `currentAid` / `currentDeviceId` / `wsUrl` 的生成移入重试循环内部，每次 `attempt` 都重新生成随机标识和连接地址
  3. 新增 `calcMinAudioBytes(textLen)` 动态计算最小有效音频字节数：`<=5` 字 500 字节，`<=20` 字 1500 字节，否则 `max(2000, textLen * 80)`
  4. `MAX_CHARS_PER_SYNTHESIS` 从 500 提升到 800；`TIMEOUT_MS` 从 30000 提升到 45000
  5. 重试间隔改为指数退避 + 随机抖动：`Math.min(8000, 500 * Math.pow(2, attempt)) + Math.floor(Math.random() * 500)`
  6. 脚本/引擎版本号同步升级到 v1.10：JSON 顶层 `name` / `version`、obj0 头部注释、obj1 `name` / `version`、引擎 url code 头部注释
- **影响**:
  - 长文本切分优先保证语义完整，显著减少半句话、跳段、音频截断现象
  - 每次重试使用全新连接标识，降低因单标识限流/连接复用冲突导致的连续失败
  - 动态最小字节数过滤掉更多截断/异常音频，避免被误判为成功
  - 单次合成容量和超时阈值更宽松，提升长句/复杂网络环境下的稳定性
- **当前文件**:
  - `new/(脚本)猫剪豆问（自然情绪版）v1.10.json`
  - `new/猫剪豆问（自然情绪版）v1.10.json`
