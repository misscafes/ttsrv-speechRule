# 猫剪豆问 v1.7 脚本 + v1.4 引擎逻辑说明

> 本文档介绍当前最新版本的猫剪豆问朗读脚本（v1.7）和 TTS 引擎（v1.4）的核心逻辑与协作方式。

## 一、文件定位

| 文件 | 定位 | 作用 |
|------|------|------|
| `new/(脚本)猫剪豆问（优化版）v1.7.json` | 朗读脚本 | 文本分析、角色识别、发音人标注、情绪桥接 |
| `new/猫剪豆问（优化版）v1.4.json` | TTS 引擎 | 接收标注后的文本，调用猫箱 API 合成语音 |

两者是"前后道工序"关系：脚本先处理文本，引擎后合成语音。

---

## 二、v1.7 脚本核心逻辑

### 1. 发音人配置

脚本内置两类角色：

- **`MAIN_ROLES_CONFIG`**：主角色配置（当前为空，由外部列表决定）
- **`BATCH_ROLES`**：批量生成的通用发音人
  - 男主、女主：各 20 个
  - 少女/少年/女青年/男青年/女中年/男中年/女老年/男老年/女童/男童：各 300 个
- **`SPECIAL_ROLES`**：特殊场景发音人
  - 括号1：【】括号内容
  - 括号2：在线音效
  - 括号3：「」括号内容
  - 括号4：『对话旁白』

### 2. 角色启用过滤

脚本读取外部文件 `jiaoseliebiao-list.json`，根据其中 `speechRule.tag` 决定启用哪些发音人。如果读取失败，默认启用所有内置发音人。

启用后的发音人会写入 `fayinren.json`，按自然数顺序排序（如 `少女01`~`少女300`，而非字典序）。

### 3. 文本处理流程

每段文本进入脚本后，按以下顺序处理：

1. **异常引号处理**：只拦截"有右引号但无左引号"的特殊段落
2. **提取对话**：按中文双引号提取 `"..."` 对话
3. **无引号文本匹配**：尝试用章节缓存匹配多行无引号对话
4. **章节缓存匹配**：优先用进度指针顺序命中，失败则全文定位
5. **AI 分析**：缓存未命中时调用大模型分析说话人
6. **发音人分配**：根据性别/年龄分配发音人，同一角色固定声音
7. **角色记录保存**：写入 `characterRecords.json`
8. **情绪桥接**：给对话和旁白添加 `[[emo:xxx]]` 前缀

### 4. 情绪桥接逻辑

情绪桥接是脚本最后一个对象（obj2.js），负责给文本加情绪标记。

#### 对话情绪
- 本地关键词推断强情绪
- 无强情绪时继承上一段情绪
- 在左引号后插入 `[[emo:xxx]]`

#### 旁白情绪
- 本地关键词推断强情绪
- 无强情绪时继承上一段情绪
- 仍无情绪时使用默认兜底：`DEFAULT_NARRATION_EMOTION = "平静"`
- 在旁白开头插入 `[[emo:xxx]]`

## 三、v1.4 引擎核心逻辑

### 1. 文本预处理

引擎接收脚本处理后的文本，执行：
- 去掉 `【】`、`『』` 等特殊括号
- 极短对话（1~15 汉字）去掉引号
- 识别并移除音效文本

### 2. 分段

引擎按中文双引号把文本切成对话段和旁白段：
- 引号内：对话
- 引号外：旁白

### 3. 发音人匹配

对话中的 `<<voiceTag>>` 被提取后，到 `tagConfig` 中查找对应配置。`tagConfig` 从 `jiaoseliebiao-list.json` 加载。

如果找不到对应配置，使用默认发音人 `zh_female_vv_uranus_bigtts`。

### 4. 情绪消费

#### 对话情绪
- 提取 `[[emo:xxx]]` 标记
- 移除标记，避免被朗读
- 保存情绪值到 `roleCfg.emotion`

#### 旁白情绪（v1.3 关键改动）
- 提取 `[[emo:xxx]]` 标记
- 移除标记
- **直接保存情绪值到 `narCfg.emotion`**
- 不再要求 narrator voice ID 含 `emo` 字样

### 5. 合成参数

合成音频时，只要 segment 配置中有 `emotion`，就传入：

```js
extraObj.audio_config.emotion = cfg.emotion;
extraObj.audio_config.emotion_scale = 4;
```

### 6. 合成重试与音频校验（v1.4 关键改动）

针对 502 错误导致的跳段，以及音频播放一半就切下一句的问题，v1.4 引擎增强了合成请求重试逻辑：

#### 触发重试的情况
- 抛出异常且包含 502/504/timeout/connection/socket 等可重试关键字
- `ws.maoxiang` 返回 `null`
- 返回音频长度为 0（空音频）
- 返回音频字节数小于 200（过短音频，疑似被截断）

#### 重试策略
- 最大重试次数：**5 次**
- 重试间隔递增：500ms、900ms、1300ms、1700ms、2100ms
- 每次重试都会记录日志，包含失败原因、已重试次数、文本前 25 字

#### 失败处理
- 5 次重试后仍失败：记录错误日志并**跳过本段**，继续合成后续段落
- 合成成功：记录音频字节数、voice、文本前 25 字，便于排查音频截断问题

#### 额外修复
- 修复了 `segments.length === 0` 兜底分支中 `allTrim` 只移除普通引号、未移除中文引号 `“”` 的 bug

## 四、脚本与引擎协作流程

```
原始文本
    ↓
v1.7 脚本
    ├── 识别对话/旁白
    ├── 分析说话人
    ├── 分配发音人（<<voiceTag>>）
    └── 添加情绪标记（[[emo:xxx]]）
    ↓
标注后文本
    ↓
v1.4 引擎
    ├── 提取 <<voiceTag>> 匹配发音人配置
    ├── 提取 [[emo:xxx]] 并移除标记
    ├── 合成音频（带 emotion 参数）
    └── 失败时自动重试 5 次，仍失败则跳过本段
    ↓
音频输出
```

## 五、关键外部依赖

| 文件 | 作用 |
|------|------|
| `jiaoseliebiao-list.json` | 发音人配置列表，决定启用哪些 voice |
| `fayinren.json` | 脚本生成的已启用发音人列表 |
| `characterRecords.json` | 当前书籍的角色记录（名字、别名、性别、年龄、发音人） |
| `miyue.txt` | AI API 密钥配置 |
| `data.json` | Legado 提供的当前书籍内容 |
| `ttsrv-replaces4.json` | 音效/文本替换规则 |
