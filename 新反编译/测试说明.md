# 命无言_缓存日志版 闪退排查说明

## 已发现的问题

1. **资源引用错误（已修复）**
   - 原 `命无言_缓存日志版.apk` 中，`dr/i.smali` 引用了一个不存在的资源 ID `0x7f0c00eb`
   - 同时引用了不存在的类 `Ldr/j;`
   - 这些问题在运行到特定 RecyclerView 时会导致崩溃

2. **中文路径导致 apktool 回编失败**
   - 在当前中文目录 `命无言_decoded_mod` 下，`apktool b` 会因编码问题无法回编
   - 我已将目录复制到英文临时路径后回编成功

3. **签名与原版不同**
   - 原 APK 使用 `CERT.RSA` 签名
   - 重打包后的 APK 使用 debug 签名 `ANDROIDD.RSA`
   - 如果手机上已安装原版，需要先卸载才能安装测试版

## 测试步骤

请按顺序测试，每步都记录结果：

### 步骤 1：测试原始重打包版

安装 `命无言_原始重打包测试版.apk`，**不要做任何修改**，看能否正常打开。

- **如果能打开**：说明 apktool 反编译/回编本身没问题，问题出在你的代码/资源修改上。继续步骤 2。
- **如果打不开/闪退**：说明 apktool 回编过程本身有问题，或签名/设备兼容性导致。请抓日志分析。

### 步骤 2：测试缓存日志修复版

安装 `命无言_缓存日志修复版.apk`，看能否正常打开。

- 这个版本已修复 `0x7f0c00eb` 资源引用错误和 `Ldr/j` 类缺失问题。
- **如果能打开**：说明主要问题已解决。
- **如果仍然闪退**：说明还有其他原因，请抓日志。

### 步骤 3：抓取崩溃日志

如果仍然闪退，请连接手机并运行 `抓取崩溃日志.bat`，然后打开 App 触发崩溃，最后把生成的 `crash_log.txt` 发给 AI 分析。

操作前提：
1. 手机开启 USB 调试（设置 → 开发者选项 → USB 调试）
2. 用 USB 线连接电脑
3. 在电脑上双击运行 `抓取崩溃日志.bat`
4. 在手机上打开 App，触发闪退
5. 按 Ctrl+C 停止脚本
6. 把 `crash_log.txt` 内容发给 AI

## 注意事项

- 测试前建议先卸载手机上已有的同名应用，避免签名冲突导致安装失败
- 两个测试版都使用 debug 签名，仅供测试，不能覆盖安装原版
- 如果测试版能正常打开，后续需要用你的正式签名密钥重新签名才能作为正式版使用
